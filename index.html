1 LZ78 (Ziv and Lempel, 1978)

  • In 1978, Ziv and Lempel published the LZ78 algorithm [1].
  • LZ78 represents the dictionary in a recursive way with the idea of reducing the memory used for representing the strings in the dictionary. Now, each entry in the dictionary is a pair wk, where w is an index to an entry of the dictionary and k is a symbol. In fact, each pair wk represents the string that results from the concatenation of the string w and the symbol k, where w can be recursively computed in the same way we have found wk.
  • We will denote the string that w represents by string(w).
  • The empty string is obtained by string(0).

2 Encoder

  1. w ← 0.
  2. While the input is not exhausted:
    1. k ← next input symbol.
    2. If wk is found in the dictionary, then:
      1. w ← address of wk in the dictionary.
    3. Else:
      1. Output wk.
      2. Insert wk in the dictionary.
      3. w ← 0.

3 Example

4 Decoder

  1. While the input is not exhausted:
    1. Input wk.
    2. Output string(w).
    3. Output k.
    4. Insert wk in the dictionary.
